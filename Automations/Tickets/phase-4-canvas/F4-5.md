# F4-5 Primer: Live Run Highlighting

**For:** New coding-agent session
**Project:** OpenHuman — Workflows & Automations
**Previous work:** F4-1..F4-4 ship the canvas. F-8 emits `WorkflowRunStepStarted` / `WorkflowRunStepCompleted` events.

---

## What Is This Ticket?

A node on the canvas lights up (per OQ-17: colored border + step-time tooltip) while it executes; flips to green/red on completion. Lets the user watch their workflow run in real time.

### Why It Matters

- The demo moment Phase 4 unlocks. "I built this in chat, here it runs" with the canvas showing the agent walking the graph.

---

## What This Ticket Must Accomplish

### Goal

- Subscribe to the socket-relay's `WorkflowRunStepStarted` / `WorkflowRunStepCompleted` / `WorkflowRunCompleted` events on the canvas page.
- For each event, map to the canvas node (by `node_id`) and update its state (`pending` / `running` / `succeeded` / `failed` / `cancelled` / `skipped`).
- Render state as a colored border + small chip in the corner of the node.
- "Watch live runs" toggle in the canvas header. Default ON.
- Show an active-run banner at the top with the run's elapsed time + a [Cancel run] button (calls `workflows_cancel_run`).
- Run history accessible via a "Recent runs" sidebar tab inside the canvas drawer.

### Deliverables Checklist

#### A. Implementation

- [ ] Socket event subscription via the existing `socketService`.
- [ ] Per-node state machine + render.
- [ ] Active-run banner.
- [ ] Recent-runs tab using `workflows_list_runs`.

#### B. Tests

- [ ] Simulate a stream of events → nodes update colors in order.
- [ ] Cancel button calls `workflows_cancel_run`.
