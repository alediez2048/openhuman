# F4-2 Primer: Node Palette Sidebar

**For:** New coding-agent session
**Project:** OpenHuman — Workflows & Automations
**Previous work:** F4-1 ships the read-only canvas.

---

## What Is This Ticket?

Sidebar with one drag-source pill per `NodeKind`. Drag onto the canvas creates a new node at the drop position.

### Why It Matters

- The first edit affordance. Without this, the canvas is decoration.

---

## What This Ticket Must Accomplish

### Goal

- New `<NodePalette>` component on the left edge of `WorkflowCanvas.tsx`.
- One drag-source per `NodeKind` (Phase 1 + Phase 2 + Phase 4 kinds: AgentPrompt, ToolCall, HttpRequest, ChannelMessage, Condition, Delay, Transform, AwaitHumanApproval, FanOut).
- Each pill: icon + label + 1-line description.
- HTML5 drag-and-drop (no third-party). On drop:
    - Append a new `Node` to the local workflow state with a fresh UUID, the dragged kind, and a default `NodeConfig` for that kind.
    - Re-layout (or honor the drop position).
- Local state only; saving back to the server is F4-9.

### Deliverables Checklist

#### A. Implementation

- [ ] `<NodePalette>` with the 9 entries.
- [ ] Drop handler on the canvas pane.
- [ ] Default-config factory per kind (sensible empty values: `AgentPromptConfig { prompt: "", iteration_cap: 12, allowed_connections: [], model_tier: None }`, `DelayConfig { duration_secs: 60 }`, etc.).

#### B. Tests

- [ ] Drag-drop of each kind appends the right `NodeConfig`.
- [ ] Default-config factories produce shapes that pass `validate(phase: 3)`.
