# F4-6 Primer: `transform` Node Kind

**For:** New coding-agent session
**Project:** OpenHuman — Workflows & Automations
**Previous work:** Phase 2 ships every other node kind. `NodeKind::Transform` already declared in F-1.

---

## What Is This Ticket?

JSONPath / jq-style shaping between nodes. Lets the user reshape one node's output into the next node's input without an LLM step.

### Why It Matters

- Closes the "I need to pluck the email subject out of the JSON before sending it to Slack" pattern that's currently a wasted `agent_prompt` step.

---

## What This Ticket Must Accomplish

### Goal

- `TransformConfig`:
    ```rust
    pub struct TransformConfig {
        pub source: String,           // template: "{{node.n1.output}}"
        pub spec: TransformSpec,      // JsonPath { path: "$.subject" } or Jq { query: ".subject" }
        pub output_format: TransformFormat,  // Json | Text
    }
    ```
- Executor body:
    1. Substitute `source` against `NodeContext`.
    2. Apply `spec` (use `serde_json_path` for JsonPath; `jaq` for jq).
    3. Return `NodeOutput { text, json }` per `output_format`.
- Validator: parse `spec.path` / `spec.query` at validate time.
- Per-form on the canvas: live preview against a sample input.

### Deliverables Checklist

#### A. Implementation

- [ ] `TransformConfig`, `TransformSpec`, `TransformFormat`.
- [ ] `dispatch_node` arm.
- [ ] Choose `serde_json_path` for JsonPath (already in tree? if not, add).
- [ ] Choose `jaq` or `jq-rs` for jq mode. Bench latency; the executor stays sub-50 ms per node.
- [ ] Canvas form with live preview.

#### B. Tests

- [ ] JsonPath `$.subject` extracts the field.
- [ ] jq `.items | length` counts items.
- [ ] Malformed path → validator error.
- [ ] Templated source: previous node's output flows into transform.
