# F3-3 Primer: Per-Node Config Drawer

**For:** New coding-agent session
**Project:** OpenHuman — Workflows & Automations
**Previous work:** F3-1 + F3-2 ship the canvas + palette.

---

## What Is This Ticket?

Click a node on the canvas → a sidebar drawer opens on the right showing that node's editable config. Per OQ-16 the drawer pushes the canvas left rather than overlaying it.

### Why It Matters

- Without this, the user can drop nodes but can't actually configure them.

---

## What This Ticket Must Accomplish

### Goal

- New `<NodeConfigDrawer>` — slides in from the right (~360px wide) when a node is selected.
- Renders the right form per `NodeKind`:
    - `AgentPrompt`: prompt textarea + iteration cap number + connections multi-select.
    - `ToolCall`: tool-name select + args JSON editor (use `<JsonEditor>` if it already exists).
    - `HttpRequest`: connection picker + method dropdown + path + headers + body + response_capture.
    - `ChannelMessage`: channel ref picker + body template.
    - `Condition`: left/op/right + then/else node-id pickers.
    - `Delay`: seconds input (capped at 86400).
    - `Transform`: JSONPath / jq snippet editor.
    - `AwaitHumanApproval`: prompt label + timeout.
    - `FanOut`: child-node list + concurrency cap.
- Each form lives in `app/src/components/workflows/canvas/forms/<Kind>Form.tsx`.
- Save updates the local canvas state — persistence is F3-9.
- Delete node button at the drawer footer (also deletes any incoming/outgoing edges).

### Deliverables Checklist

#### A. Implementation

- [ ] `<NodeConfigDrawer>` with `NodeConfig` discriminator switch.
- [ ] 9 per-kind form components.
- [ ] Drawer open/close animation respects `prefers-reduced-motion`.

#### B. Tests

- [ ] Vitest per form: type into a field → emits the right `NodeConfig` partial.
- [ ] Delete button removes the node + connected edges.
