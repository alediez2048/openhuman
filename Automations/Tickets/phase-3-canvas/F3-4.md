# F3-4 Primer: Edge Wiring + DAG Validation

**For:** New coding-agent session
**Project:** OpenHuman — Workflows & Automations
**Previous work:** F3-1..F3-3 ship the canvas + palette + drawer. F2-2 already supports DAGs via `topological_sort`.

---

## What Is This Ticket?

Let the user draw + delete edges between nodes on the canvas. The Phase 2 executor already walks DAGs; F3-4 unlocks the UI to author them.

### Why It Matters

- Phase 1 + 2 workflows are linear by construction (catalog + drafter). The canvas is where the user authors non-linear graphs.

---

## What This Ticket Must Accomplish

### Goal

- xyflow's edge-creation handles already exist — wire them to update the local workflow `edges[]`.
- Cycle detection on every edge drop: re-run `topological_sort` client-side (small reimplementation of F2-2's check) and reject the edge with an inline toast on cycle.
- Edge delete via click + Delete key.
- Validator already rejects dangling `from` / `to` (F-11); the canvas just shouldn't let the user create them.
- Special-case the `Condition` node: it has two outbound edges (then / else). The UI surfaces both handles with labels.
- Special-case the `FanOut` node: outbound edges go to children that run in parallel; surface as a "fan-out" handle.

### Deliverables Checklist

#### A. Implementation

- [ ] xyflow `onConnect` handler → append to `edges[]`.
- [ ] Client-side `detectCycle(nodes, edges, newEdge)` helper.
- [ ] Multi-handle node renderer for `Condition` + `FanOut`.
- [ ] Edge label rendering (the condition's "then" / "else" labels).

#### B. Tests

- [ ] Drag from node A's outlet to node B's inlet → edge appears in `edges[]`.
- [ ] Drag a cycle → rejected with toast; `edges[]` unchanged.
- [ ] Delete an edge via click + Delete.
- [ ] Condition node renders both handles + edge labels.
