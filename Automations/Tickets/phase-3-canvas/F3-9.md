# F3-9 Primer: Canvas-Driven Create Flow

**For:** New coding-agent session
**Project:** OpenHuman — Workflows & Automations
**Previous work:** F3-1..F3-8 ship the canvas for editing existing workflows.

---

## What Is This Ticket?

A "Build with canvas" alternative to the chat-driven creation flow. Same `/workflows/new` entry point; the user picks Chat (default) or Canvas. Per OQ-20, chat stays primary; canvas is the power-user fallback.

### Why It Matters

- Closes the loop. F3-1..F3-8 let you edit existing workflows visually but didn't expose a way to create from scratch on the canvas.

---

## What This Ticket Must Accomplish

### Goal

- `/workflows/new` route with two tabs:
    - **Chat** (existing) — navigates to `/chat` with a primer message.
    - **Canvas** — opens a blank `WorkflowCanvas` with a placeholder Manual trigger + a single empty AgentPrompt node.
- Save flow:
    - "Save" button in the canvas header.
    - On click: build the `CreateWorkflowRequest` from local state, call `workflows_create` with `origin: { type: 'user_form' }` (matches FR-1.1.6's reserved variant for the visual builder).
    - On success: navigate to `/workflows/<id>/canvas` for further editing.
- "Save & Enable" variant calls `workflows_enable` after `create`.
- Discard returns to `/workflows`.
- Validation runs client-side via the F-11 validator's shape (port the validation rules to TS, OR call a new `workflows_validate` RPC for a server-side check).

### Deliverables Checklist

#### A. Implementation

- [ ] `/workflows/new` route + tab switcher.
- [ ] Empty-canvas seed (Manual trigger + 1 AgentPrompt node).
- [ ] Save / Save & Enable / Discard buttons + handlers.
- [ ] Client-side validation (port from `validator.rs` OR new RPC).
- [ ] `origin: UserForm` discriminator wired through.

#### B. Tests

- [ ] Open `/workflows/new`, drag a ChannelMessage node, wire it, fill the form, Save → workflow appears in `/workflows`.
- [ ] Save & Enable enables on success.
- [ ] Discard does not persist.
- [ ] Invalid shape (e.g. dangling edge from F3-4) blocks Save with inline error.
