## What Is This Ticket?

Fill the empty `ops.rs` stub from P0-1 with the Generic HTTP connection CRUD layer, and extend `rpc.rs` / `schemas.rs` with the four matching RPC methods: `connections_generic_http_create`, `_update`, `_delete`, and `connections_test`. The `secret_ref` field on `GenericHttpConnection` integrates with the existing `src/openhuman/security/secrets.rs` store â€” credentials live there and are referenced by id, so they never round-trip through workflow JSON, log lines, or RPC responses.

`connections_test` issues a best-effort HEAD (falling back to OPTIONS, then GET) against the connection's `base_url` with the resolved auth header attached. It returns `{ ok: bool, status: Option<u16>, error: Option<String> }`. This is a "is the credential plausibly working?" probe, not a deep health check.

### Why It Matters

- Generic HTTP is the **escape hatch connection type** for Phase 2's `http_request` node (ADR-005). Without P0-3, the user has no way to register an external REST service.
- Generic HTTP also lets the user wire up n8n / Zapier / IFTTT / Make webhooks outbound (per the hybrid positioning ADR-005).
- The `security/secrets.rs` integration is the *security boundary* (NFR-2.3.2): credentials must never appear in workflow JSON, run records, or log lines. P0-3 establishes the discipline.
- `connections_test` gives the UI an immediate "did I configure this right?" affordance â€” critical for first-time use.

## Architectural Decisions

| Decision | Reference | Summary |
|---|---|---|
| Credentials live in `security/secrets`, referenced by id | NFR-2.3.2 / ADR-006 | Workflow JSON, run records, and logs only carry a `SecretRef`, never the cleartext |
| `connections_test` is best-effort, not deep | requirements Â§1.10 | A 200/204/301/etc. is "good enough"; failures return a structured result, not a panic |
| HEAD â†’ OPTIONS â†’ GET fallback | this ticket | Some servers reject HEAD with 405; OpenHuman shouldn't paper over that as "broken" |
| Secret-write order: secret first, row second, event last | ops.rs rustdoc | An orphan secret is recoverable; an orphan row pointing at a missing secret is broken |
| Secret-delete order: row first, secret second | ops.rs rustdoc | An orphan secret is recoverable on next prune; a row pointing at a missing secret is broken |
| 10-second probe timeout | this ticket | UX bound â€” the user is waiting for the "test" button |

## Suggested Implementation Pattern

**`src/openhuman/connections/ops.rs` (extract):**

```rust
pub async fn create_generic_http(
    ctx: &Ctx,
    req: CreateGenericHttpRequest,
) -> anyhow::Result<GenericHttpConnection> {
    // Step 1: write the secret (if any) BEFORE the row.
    let secret_ref = match req.auth_credential {
        Some(cred) => Some(ctx.secrets().put(cred.into_secret_payload()).await?),
        None => None,
    };

    // Step 2: persist the row.
    let conn = GenericHttpConnection {
        id: Uuid::now_v7().to_string(),
        name: req.name,
        base_url: req.base_url,
        auth_kind: req.auth_kind,
        secret_ref,
        default_headers: req.default_headers,
        created_at: Utc::now(),
        updated_at: Utc::now(),
    };
    ctx.connections_store().insert_generic_http(&conn)?;

    // Step 3: announce.
    publish_global(DomainEvent::ConnectionAdded {
        r#ref: ConnectionRef::GenericHttp { connection_id: conn.id.clone() },
    });

    tracing::info!(target: "connections", "[connections] generic_http created id={}", conn.id);
    Ok(conn)
}

pub async fn test_generic_http(ctx: &Ctx, id: &str) -> anyhow::Result<TestProbeResult> {
    let conn = ctx.connections_store().get_generic_http(id)?;
    let header = resolve_auth_header(&conn, ctx).await?;
    let client = ctx.http_client_factory().build()?;

    for method in [reqwest::Method::HEAD, reqwest::Method::OPTIONS, reqwest::Method::GET] {
        let mut req = client.request(method.clone(), &conn.base_url);
        if let Some((name, value)) = header.as_ref() {
            req = req.header(name, value);
        }
        match tokio::time::timeout(Duration::from_secs(10), req.send()).await {
            Ok(Ok(resp)) if resp.status().as_u16() == 405 && method == reqwest::Method::HEAD => continue,
            Ok(Ok(resp)) => {
                let s = resp.status().as_u16();
                return Ok(TestProbeResult { ok: s < 400, status: Some(s), error: None });
            }
            Ok(Err(e))  => return Ok(TestProbeResult { ok: false, status: None, error: Some(format!("{e}")) }),
            Err(_)      => return Ok(TestProbeResult { ok: false, status: None, error: Some("timeout".into()) }),
        }
    }
    Ok(TestProbeResult { ok: false, status: Some(405), error: Some("HEAD/OPTIONS/GET all rejected".into()) })
}
```

## Edge Cases to Handle

1. **`auth_kind = None`.** No credential needed; `secret_ref` stays `None`; `resolve_auth_header` returns `Ok(None)`; `connections_test` sends no auth header.
2. **`auth_kind = Bearer` with empty token.** Validate at `create_generic_http`: empty string â†’ return `RpcOutcome::Err(ValidationError { field: "auth_credential" })`. Don't persist a useless row.
3. **`base_url` without a scheme.** Validate: must start with `http://` or `https://`. Reject otherwise.
4. **`base_url` with a trailing slash.** Normalize: strip exactly one trailing slash before persisting, so probe builds a clean URL.
5. **Secret rotation on update where the new credential is identical to the old.** Don't write a redundant secret â€” compare and skip the secret update; only the row's `updated_at` bumps.
6. **`update_generic_http` where the row doesn't exist.** Return a structured `RpcOutcome::Err(NotFound { id })`; do NOT write a secret.
7. **Probe response is a redirect chain ending in 200.** The default `reqwest` client follows redirects up to 10 hops â€” keep that behavior; redirects are normal for many APIs.
8. **Probe against `localhost` or RFC1918 ranges.** Allow it for Phase 0 â€” the user explicitly registered the connection. (SSRF mitigation is a Phase 2 concern when the agent calls `http_request` autonomously.)
9. **Concurrent create with the same `name`.** No uniqueness constraint on `name`. Two rows with identical names are legal; the UI's deduplication is by `id`.
10. **`security::secrets::put` fails mid-create.** Bubble the error; row was not yet written, so state stays consistent.
