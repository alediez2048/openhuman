# P0-4 Primer: `/skills` → `/connections` route rename + `/channels` redirect

**For:** New coding-agent session
**Project:** OpenHuman — Workflows & Automations
**Date:** 2026-05-19
**Previous work:** [`P0-3.md`](./P0-3.md) — Generic HTTP CRUD + secrets integration. The Rust core now exposes everything the UI needs.

---

## What Is This Ticket?

Pure frontend refactor. Rename `app/src/pages/Skills/` → `app/src/pages/Connections/` (and the `Skills.tsx` shim file), update `app/src/AppRoutes.tsx` so the canonical route is `/connections`, and add `/skills` + `/channels` → `/connections` redirects. Update i18n keys, bottom-tab nav `path`, component imports, and any tests that referenced the old path.

**No new UI sections in this ticket.** The existing Composio + Skills page contents move verbatim from `/skills` to `/connections`. The "Connections Hub" expansion (6-section layout, new sections for Built-in / MCP / Generic HTTP) lands in P0-5 and P0-6.

**No backend changes.** All Phase 0 backend work is complete (P0-1 + P0-2 + P0-3).

### Why It Matters

- The rename is the smallest possible commit that establishes the new vocabulary across the codebase. Splitting it from the larger UI build (P0-5) keeps the diff reviewable.
- Keeping `/skills` and `/channels` as 301-equivalents (HashRouter `<Navigate>`) preserves muscle memory, deep links the user has saved, and any external docs referencing the old paths.
- Sets up P0-5 to build the Connections Hub layout in a clean directory without simultaneous path migration noise.

## Edge Cases to Handle

1. **A deep link with a query string: `/skills?tab=composio`.** `<Navigate to="/connections" replace />` does not propagate the query string by default. Verify with a test; if needed, use a small component that `useLocation()`s and rebuilds the target URL.
2. **An external doc link to `/channels`.** The redirect should land on `/connections#channels`; P0-5 will scroll-to-section on that hash. For P0-4, just verify the URL lands at the right page; no scroll yet.
3. **A user with a long-running Skills page open during an update.** HashRouter reloads safely; nothing to do.
4. **Walkthrough/Joyride targets that selector by tab id.** Search `app/src/components/walkthrough/` for `'skills'` literals tied to the bottom-tab id; update to `'connections'`.
5. **Analytics events fired with `route: '/skills'`.** Optional cleanup — log the redirect once at startup if the user lands on the old path. Not required for this ticket.
6. **Mock backend tests that mount the Skills page directly.** If any Vitest test imports `Skills` from `./pages/Skills`, update to `Connections` from `./pages/Connections`.
7. **CSS imports keyed off the old path.** Unlikely with Tailwind, but search for `[data-route="/skills"]` selectors.
