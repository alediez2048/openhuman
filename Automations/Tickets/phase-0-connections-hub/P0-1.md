## What Is This Ticket?

Create the `src/openhuman/connections/` Rust domain — the foundation for the Phase 0 Connections Hub. This ticket establishes the module layout, the core types (`ConnectionRef`, `GenericHttpConnection`, `ConnectionView`, `AuthKind`, `ConnectionStatus`), the SQLite store at `${OPENHUMAN_WORKSPACE}/connections.db` with its first migration, and registers the empty controller skeleton so future tickets can plug in handlers.

**Crucially: no RPC handlers are wired up in this ticket** — only the scaffolding and the `connections.db` migration. P0-2 adds the `connections_list` RPC; P0-3 adds the Generic HTTP CRUD.

### Why It Matters

- Every subsequent Phase 0 ticket (P0-2 through P0-7) depends on this scaffolding existing.
- The two-database architecture (`connections.db` + `workflows.db`) is locked by ADR-003. Creating `connections.db` correctly here means Phase 1 doesn't have to retrofit the file boundary.
- Establishes the precedent for the `workflows/` domain in Phase 1 — same module layout, same migration pattern.

## Edge Cases to Handle

1. **Workspace dir doesn't exist.** `ConnectionsStore::open` must create parent directories or return a clear error. Match the cron store's behavior.
2. **Migration partial failure.** SQLite transactions wrap each migration. A failed migration leaves the DB unchanged so retry on next startup works.
3. **`schema_migrations` table doesn't exist yet on first boot.** `apply_migrations` must `CREATE TABLE IF NOT EXISTS schema_migrations` before checking for applied versions.
4. **Concurrent core startups against the same workspace.** SQLite handles this with file-level locking; the open call may block briefly. Not a Phase 0 concern, but don't break the convention.
5. **Tests on Windows.** Use `tempfile::TempDir` and a `Path` rather than hard-coded `/tmp/…` paths.
6. **`SecretRef` shape.** The existing `src/openhuman/security/secrets/` module defines what a `SecretRef` looks like — re-use that type, don't redefine it.
