# P0-2 Primer: `connections_list` RPC + aggregator over existing mechanisms

**For:** New coding-agent session
**Project:** OpenHuman — Workflows & Automations
**Date:** 2026-05-19
**Previous work:** [`P0-1.md`](./P0-1.md) — Rust domain skeleton, `connections.db` migration, empty controller scaffold.

---

## What Is This Ticket?

Fill the empty `aggregator.rs`, `rpc.rs`, `bus.rs`, and `schemas.rs` stubs left by P0-1 with the first real read surface for the Connections domain. The aggregator walks all 5 existing connection mechanisms — `composio/`, `channels/`, `webview_accounts/`, `integrations/`, and the MCP server registry — plus the new `generic_http_connections` table, and produces a single `Vec<ConnectionView>`. This list is exposed via the new JSON-RPC method `connections_list`. The event-bus wiring publishes `ConnectionAdded` / `ConnectionRemoved` / `ConnectionUpdated` events so the Phase 1 workflow-health subscriber (P1-3, not in this phase) can recompute health reactively.

**No frontend yet.** This ticket is pure Rust. P0-5 will call `connections_list` from the React page.

### Why It Matters

- `connections_list` is the foundational read API for the entire Connections Hub UI (P0-5/P0-6) and for the drafting sub-agent's hybrid connection-discovery prompt (Phase 1, OQ-13).
- The aggregator is the *single source of truth* about what the user has connected. Without it, the UI would have to fan-out to five different backends.
- Publishing `ConnectionAdded` / `ConnectionRemoved` events here means the Phase 1 workflow-health recomputation (per ADR-017) can subscribe without retrofitting every per-mechanism domain.

## What This Ticket Must Accomplish

### Goal

Land a working `connections_list` JSON-RPC method that returns the union of all 6 connection mechanisms as a flat `Vec<ConnectionView>` — Composio toolkits + connected accounts, native chat channels, webview accounts, built-in integrations (Twilio/Apify/Google Places/Parallel/Seltz/Stock Prices), MCP servers, and generic-HTTP connections. The aggregator is read-only against the existing per-mechanism stores. Event-bus subscribers in `bus.rs` are wired to forward per-mechanism connection lifecycle events into the unified `DomainEvent::ConnectionAdded/Removed/Updated` stream.

## Architectural Decisions

| Decision | Reference | Summary |
|---|---|---|
| Aggregator reads through per-mechanism public APIs | ADR-003 | Keeps `connections.db` only as the source of truth for Generic HTTP; everything else stays in its home domain |
| Per-collector errors degrade gracefully | systemsdesign §3.2 (executor pattern) | A broken Composio cache must not blank the entire Hub page |
| Lifecycle events republished as unified `ConnectionAdded`/`Removed`/`Updated` | ADR-017 | Phase 1's workflow-health subscriber subscribes once, not five times |
| `connections_list` is the only read RPC in P0-2 | ADR-006 | Phase 0 ships independently; CRUD lands in P0-3 |
| Filter + search applied post-aggregation | requirements §1.9 | N is small (`<200` typical); filtering server-side keeps the wire payload minimal |

## Suggested Implementation Pattern

**`src/openhuman/connections/aggregator.rs` (extract):**

```rust
pub async fn list_all(ctx: &Ctx) -> anyhow::Result<Vec<ConnectionView>> {
    let (composio, channels, webview, builtin, mcp, generic_http) = tokio::join!(
        collect_composio(ctx),
        collect_channels(ctx),
        collect_webview(ctx),
        collect_builtin(ctx),
        collect_mcp(ctx),
        collect_generic_http(ctx),
    );

    let mut all = Vec::new();
    for (label, result) in [
        ("composio", composio), ("channels", channels), ("webview", webview),
        ("builtin", builtin),   ("mcp", mcp),           ("generic_http", generic_http),
    ] {
        match result {
            Ok(mut rows) => all.append(&mut rows),
            Err(e) => tracing::warn!(target: "connections", "[connections] {label} collector failed: {e:#}"),
        }
    }
    all.sort_by(|a, b| (a.kind, &a.label).cmp(&(b.kind, &b.label)));
    Ok(all)
}

async fn collect_composio(ctx: &Ctx) -> anyhow::Result<Vec<ConnectionView>> {
    let toolkits = composio::ops::list_connected_toolkits(ctx).await?;
    Ok(toolkits.into_iter().map(|tk| ConnectionView {
        r#ref: ConnectionRef::Composio { toolkit_id: tk.id.clone(), account_id: tk.account_id.clone() },
        kind: ConnectionKind::Composio,
        label: tk.display_name,
        status: if tk.connected { ConnectionStatus::Connected } else { ConnectionStatus::NotConnected },
        // ... icon_url, scopes, last_synced
    }).collect())
}
```

**`src/openhuman/connections/rpc.rs` (extract):**

```rust
pub async fn handle_connections_list(
    ctx: &Ctx,
    req: ConnectionsListRequest,
) -> RpcOutcome<ConnectionsListResponse> {
    let mut connections = aggregator::list_all(ctx).await
        .map_err(|e| { tracing::error!(target: "connections", "[connections-rpc] list_all failed: {e:#}"); e })?;

    if let Some(kinds) = req.kind_filter {
        connections.retain(|c| kinds.contains(&c.kind));
    }
    if let Some(q) = req.search {
        let q = q.to_ascii_lowercase();
        connections.retain(|c| c.label.to_ascii_lowercase().contains(&q));
    }

    RpcOutcome::Ok(ConnectionsListResponse {
        connections,
        generated_at: chrono::Utc::now(),
    })
}
```

**`src/openhuman/connections/bus.rs` (extract):**

```rust
pub struct ConnectionsLifecycleSubscriber;

#[async_trait::async_trait]
impl EventHandler for ConnectionsLifecycleSubscriber {
    fn name(&self) -> &'static str { "connection::lifecycle" }
    fn domains(&self) -> Option<&'static [&'static str]> {
        Some(&["composio", "channel", "webview_account", "integration", "mcp"])
    }

    async fn handle(&self, event: &DomainEvent) {
        let unified = match event {
            DomainEvent::ComposioAccountConnected { toolkit_id, account_id } =>
                Some(DomainEvent::ConnectionAdded {
                    r#ref: ConnectionRef::Composio { toolkit_id: toolkit_id.clone(), account_id: account_id.clone() },
                }),
            // … same shape for Channel, Webview, Builtin, Mcp
            _ => None,
        };
        if let Some(ev) = unified { publish_global(ev); }
    }
}
```

## Edge Cases to Handle

1. **A mechanism's domain is uninitialized.** E.g., `composio::ops::list_connected_toolkits` returns `Err` before the user has ever opened the Composio settings. Log and degrade — return empty for that mechanism.
2. **A webview account is in `Error { reason }` state.** Surface it as a `ConnectionView` with `status = Error { reason }`, not omitted. The UI should be able to show the broken account so the user can fix it.
3. **A Composio toolkit is configured but the OAuth token is expired.** Treat it the same as `Error { reason: "session_expired" }`.
4. **MCP server registry returns a stopped server.** Include it with `status = Disabled`. The user needs to see disabled servers to re-enable them.
5. **Built-in integration is enabled in config but has no credentials.** Include with `status = NotConnected` and a hint in a follow-up field (label suffix is fine for P0-2; richer affordances arrive in P0-6).
6. **Lifecycle event has no equivalent in the unified taxonomy.** Log at `debug` and skip — do NOT panic. Forward compat for future per-mechanism events.
7. **Empty workspace.** `connections_list` returns `connections: []`, not `Err`. The UI's empty state handles this.
