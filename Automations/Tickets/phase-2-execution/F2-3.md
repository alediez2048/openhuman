# F2-3 Primer: `tool_call` Node Kind

**For:** New coding-agent session
**Project:** OpenHuman — Workflows & Automations
**Previous work:** F2-2 shipped multi-node execution + the `NodeContext` shape.

---

## What Is This Ticket?

Implement the `tool_call` node — invoke a single named tool from `tools::registry` with literal args (templated via `NodeContext`). The simplest Phase 2 node kind; lands the executor pattern the other 4 follow.

### Why It Matters

- Unlocks deterministic single-tool steps inside workflows ("send a Slack message", "create a Composio action") without spinning up the LLM-driven `agent_prompt` overhead.

---

## What This Ticket Must Accomplish

### Goal

- `ToolCallConfig { tool_name: String, args: serde_json::Value }`.
- Executor body:
    1. Template `args` against `NodeContext` (substitutes `{{trigger.X}}` / `{{node.Y.output.Z}}`).
    2. Look up the tool in `tools::registry::list_tools()` by name.
    3. Reject if not present (return `NodeOutput::error("tool not registered: X")`).
    4. Call `tool.execute(templated_args).await`.
    5. Map `ToolResult` → `NodeOutput { text, json: ToolResult.output_for_llm }`.
- Validator: `ToolCallConfig.tool_name` must exist in the registry at validation time. Catalog templates can reference any tool the project ships.
- Allowed-connections check stays in place (per-node `allowed_connections` walks against `ConnectionsSnapshot`).

### Deliverables Checklist

#### A. Implementation

- [ ] `dispatch_node` arm for `NodeConfig::ToolCall(cfg)`.
- [ ] `templating::substitute_json(context, &cfg.args)` extension for substituting templates inside JSON values.
- [ ] Validator arm: `tool_name` lookup against `tools::registry::list_tools()`.

#### B. Tests

- [ ] Round-trip with a `current_time` tool (no args, no templating).
- [ ] Round-trip with a templated tool: previous node emits `{"x": 7}`; this node calls a stub tool with `{ "value": "{{node.n1.output.x}}" }`; assert the substituted args.
- [ ] Unknown tool name → validation error at create time.
- [ ] Tool execution failure → `NodeOutput::error` + run halts (or continues per `on_error`).
