## What Is This Ticket?

Update the drafting sub-agent's system prompt to teach the LLM about the Phase 2 trigger + node taxonomy + the templating syntax for inter-node data passing.

### Why It Matters

- Until the prompt is updated, the LLM still generates Phase 1 single-node workflows. The Phase 2 executor + validator are useless without the drafter knowing how to use them.

## What This Ticket Must Accomplish

### Goal

Edit both copies (per F-13 dual-write convention):
- `Automations/Artifacts/prompts/workflow_builder.md` (design-time SoT)
- `src/openhuman/agent/prompts/workflow_builder.md` (bundled, byte-identical)

Add sections:

1. **Phase 2 triggers** — `webhook { target_path }`, `composio_event { toolkit, trigger_id }`, `channel_message { provider, filter? }`. One sentence per variant + a worked JSON example.
2. **Phase 2 node kinds** — `tool_call`, `http_request`, `channel_message`, `condition`, `delay`. JSON shape + worked example per kind.
3. **Inter-node templating** — explain `{{trigger.<path>}}` and `{{node.<id>.output.<path>}}`. Worked example chaining `agent_prompt` → `channel_message` with the substituted body.
4. **`on_error` policy + retry budget** — when to use `Continue` vs `Halt`. Defaults: `Halt` workflow-level, `max_attempts = 1` per-node.
5. **Branching shape** — `condition` node + `then_node_id` / `else_node_id`. Worked example.
6. **Updated worked examples** — replace one or two of the Phase 1 examples with Phase 2 chains demonstrating the surface.

Keep the prompt under 6 KiB. The Phase 1 prompt is ~4 KiB; Phase 2 additions target ~2 KiB.
