# F2-11 Primer: `channel_message` Trigger + Filter

**For:** New coding-agent session
**Project:** OpenHuman — Workflows & Automations
**Previous work:** F2-10 shipped Composio-event trigger. Channels domain publishes `DomainEvent::ChannelMessageReceived`.

---

## What Is This Ticket?

Fire workflows on inbound chat-channel messages matching an optional filter. Same shape as F2-10's Composio subscriber, but the filter primitives are richer because chat messages have free-form bodies.

### Why It Matters

- The "respond to a Slack mention" / "summarise every email" pattern. Highest-volume Phase 2 trigger likely.

---

## What This Ticket Must Accomplish

### Goal

- `Trigger::ChannelMessage { provider, filter: Option<MessageFilter> }` already declared.
- `MessageFilter`:
    ```rust
    pub struct MessageFilter {
        pub contains: Option<String>,      // case-insensitive substring
        pub direct_only: bool,             // DM-only (skip group)
        pub from_user: Option<String>,     // sender id
        pub regex: Option<String>,         // optional richer match
    }
    ```
- `workflows::bus::ChannelMessageSubscriber`:
    - On `ChannelMessageReceived { provider, channel_id, sender, body, is_direct }`:
        - Walk workflows with matching `provider`.
        - Apply the filter.
        - For each pass: `dispatch_run(config, wf_id, TriggerSource::ChannelMessage { message: ... })`.
    - Inject the matched message into `NodeContext.trigger_payload`.
- Validator: regex compiles if present; `from_user` non-empty string when set.

### Deliverables Checklist

#### A. Implementation

- [ ] Extend `MessageFilter` to the full shape (F-1 declared a partial stub).
- [ ] `ChannelMessageSubscriber` + register at boot.
- [ ] `workflows::store::list_workflows_matching_channel(provider)` query.
- [ ] Filter evaluator + tests.
- [ ] Validator arms.

#### B. Tests

- [ ] Message matching `contains: "URGENT"` triggers; non-matching ignored.
- [ ] `direct_only: true` skips group messages.
- [ ] `regex: "^@bot help"` matches as expected.
- [ ] Multiple workflows match the same message → fan-out.
- [ ] Invalid regex → validator error.
