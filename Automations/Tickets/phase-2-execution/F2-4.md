# F2-4 Primer: `http_request` Node Kind + Outbound Bridge

**For:** New coding-agent session
**Project:** OpenHuman — Workflows & Automations
**Previous work:** F2-3 shipped `tool_call`. Phase 0 shipped Generic-HTTP connections with encrypted credentials.

---

## What Is This Ticket?

`http_request` is the **outbound bridge to n8n / Zapier / IFTTT / Make** per ADR-005 — and to any REST service on the planet. Sends an HTTP request via a Generic-HTTP `ConnectionRef`; substitutes node context into URL / headers / body.

### Why It Matters

- Locks the hybrid-platform-interop story from ADR-005. The user can pipe any OpenHuman workflow output into an external automation tool via webhook URL.
- The drafting sub-agent's webhook escape hatch (ADR-013) generates exactly these nodes when the user describes a service OpenHuman doesn't have a native integration for.

## What This Ticket Must Accomplish

### Goal

- `HttpRequestConfig`:
    ```rust
    pub struct HttpRequestConfig {
        pub connection_id: GenericHttpConnectionId,
        pub method: HttpMethod,         // GET / POST / PUT / DELETE / PATCH
        pub path_template: String,       // appended to the connection's base_url
        pub headers_template: HashMap<String, String>,
        pub body_template: Option<String>,
        pub response_capture: ResponseCapture,  // body / status / json-path
    }
    ```
- Executor body:
    1. Load the `GenericHttpConnection` from `connections::store`.
    2. Decrypt the `secret_ref` (Phase 0 `secret_store`).
    3. Build the `Authorization` header per the connection's `AuthKind` (Bearer / Basic / ApiKeyHeader / QueryParam / None).
    4. Substitute templates against `NodeContext`.
    5. Send via `reqwest`. Honor a hard 30s timeout (NFR override-able per-node via `settings.timeout_secs`).
    6. Capture per `response_capture` — full body, status code only, or a JSON-path slice.
    7. Return `NodeOutput { text: response_body, json: parsed_response }`.
- Honor the per-node retry policy (F2-8).
- Never log the decrypted secret or the `Authorization` header (NFR-2.4.4).

### Deliverables Checklist

#### A. Implementation

- [ ] `HttpMethod`, `ResponseCapture` types.
- [ ] `dispatch_node` arm.
- [ ] Reuse Phase 0's `connections::http::send_request` helper if it exists; otherwise add a thin wrapper.
- [ ] Validator: `connection_id` must exist in `connections.db`; `method` parseable; `headers` keys ASCII.

#### B. Tests

- [ ] Round-trip against a mock HTTP server (using the existing test-server harness).
- [ ] Templating substitutes node outputs into URL + body.
- [ ] Secret-leak regression: assert the decrypted credential never reaches `tracing::*` output.
- [ ] Auth kinds: one test per AuthKind variant.
- [ ] `response_capture = JsonPath("$.id")` returns the path-extracted value.
- [ ] 5xx response surfaces as a `NodeOutput::error` and the retry policy (F2-8) gets a chance.

## Architectural Decisions

| Decision | Reference | Summary |
|---|---|---|
| Reuse Phase 0 Generic-HTTP connections | ADR-005 | One auth-credential surface, one secret store, one verification probe |
| Templating before send, not after | Inter-node passing | Substituted output is observable per-step in the run history |
| Secret never logged | NFR-2.4.4 | Sentry / tracing redact the Authorization header |
