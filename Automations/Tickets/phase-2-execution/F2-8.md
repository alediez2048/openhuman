# F2-8 Primer: `on_error` Policy + Per-Node Retry Budget

**For:** New coding-agent session
**Project:** OpenHuman — Workflows & Automations
**Previous work:** F-7 hard-coded `on_error = Halt` for Phase 1. F2-2..F2-7 added all the kinds; this ticket lands the policy enforcement they reference.

---

## What Is This Ticket?

Wire the per-workflow `on_error` policy (Halt / Continue) into multi-node execution and add per-node retry budgets with backoff. Per FR-1.6.4 (Phase 2 unlocks `Continue`) and OQ-13.

### Why It Matters

- Without `Continue`, a chain of 5 nodes that flake on node 2 fails the whole run. With `Continue`, the run records the failure + presses on.
- Per-node retries handle the transient-network case (rate-limit + 429 from `http_request`, brief Composio outage) without escalating to a workflow failure.

---

## What This Ticket Must Accomplish

### Goal

- Already declared: `WorkflowSettings.on_error: OnErrorPolicy { Halt | Continue }`.
- New `RetryPolicy`:
    ```rust
    pub struct RetryPolicy {
        pub max_attempts: u32,            // default 1 (= no retry)
        pub backoff: BackoffSpec,         // Exponential { initial_ms, max_ms }
        pub retry_on: Vec<RetryableError>,// HttpStatus5xx / Timeout / RateLimited
    }
    ```
    Attached per-node via `Node.retry_policy: Option<RetryPolicy>` — `None` = workflow's default (= `max_attempts: 1`).
- Executor changes:
    - Wrap `dispatch_node` in a retry loop respecting `retry_policy`.
    - On exhaustion: emit `WorkflowRunStepCompleted { status: Failed, attempts: N }`.
    - Honor workflow-level `on_error`:
        - `Halt`: mark run Failed, stop further nodes.
        - `Continue`: mark step Failed, advance to next node.
- Per-node retry events: `DomainEvent::WorkflowRunStepRetried { run_id, node_id, attempt }`.
- Settings validation: `max_attempts ∈ [1, 5]`; `initial_ms ∈ [100, 10_000]`; `max_ms ≤ 60_000`.

- [ ] `tool_call` node fails all 3 attempts → step Failed; workflow honors `on_error`.
- [ ] `on_error = Continue`: step 2 fails, step 3 still runs.
- [ ] `on_error = Halt`: step 2 fails, step 3 skipped, run Failed.
- [ ] Retry backoff timing: assert successive attempts respect `initial_ms` then `2 * initial_ms` then `4 * initial_ms`, capped at `max_ms`.
