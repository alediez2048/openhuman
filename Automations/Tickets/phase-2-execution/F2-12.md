**Project:** OpenHuman — Workflows & Automations
**Previous work:** F2-1..F2-11 ship every Phase 2 trigger + node kind. F-5 / F-6 ship the catalog mechanism.

---

## What Is This Ticket?

Ship 5 starter templates that exercise the Phase 2 surface end-to-end. The Phase 1 catalog had `RU-1..RU-4` (Cron + AgentPrompt only); Phase 2 adds reference workflows for the new triggers + chains.

### Why It Matters

- Catalog templates are the fastest demo path. RU-5..RU-9 are the "look what Phase 2 unlocks" surface.
- Each template is also a regression test — the executor must keep them running for the template to stay shippable.

---

## What This Ticket Must Accomplish

### Goal

Ship 5 templates at `src/openhuman/workflows/templates/`:

| ID | Trigger | Chain | What it demos |
|---|---|---|---|
| `ru-5-stripe-payment-thank-you` | `composio_event { toolkit: "stripe", trigger_id: "payment.succeeded" }` | agent_prompt → channel_message | Composio event trigger + outbound Slack |
| `ru-6-slack-mention-triage` | `channel_message { provider: "slack", filter: { contains: "@me" } }` | agent_prompt → condition → channel_message | Channel trigger + branching |
| `ru-7-github-issue-summary` | `webhook { tunnel_uuid, target_path: "/github" }` | agent_prompt → http_request | Webhook inbound + outbound HTTP |
| `ru-8-daily-sales-rollup` | `cron { expr: "0 9 * * *" }` | http_request → agent_prompt → channel_message | Cron + HTTP + agent + channel chain |
| `ru-9-zapier-bridge` | `webhook { tunnel_uuid, target_path: "/zapier-in" }` | tool_call → http_request | Pure bridge — Zapier in, Zapier out |

Each template's JSON must:
- Set `min_phase: 2`.
- Use empty `account_id` / `channel_id` as wildcards (F-15 matching rule).
- Carry rationale bullets explaining "why this design".
- Stay under 2 KiB JSON each.

### Deliverables Checklist

#### A. Implementation

- [ ] 5 JSON files at `src/openhuman/workflows/templates/ru-5-*.json` through `ru-9-*.json`.
- [ ] Each registered via `include_str!` in `templates/mod.rs` (matches F-5 pattern).
- [ ] All 5 validate against F-11's `validate` at `phase = 2`.

#### B. Tests

- [ ] Extend `templates_tests.rs`:
    - All 9 templates (RU-1..RU-9) parse + validate.
    - `template_id`s unique across the full set.
- [ ] One round-trip test per template against a stub provider (mock LLM + mock HTTP server + stub channel).
