# F2-1 Primer: Phase 2 Scaffold — Reachable NodeKind / NodeConfig / Trigger Variants

**For:** New coding-agent session
**Project:** OpenHuman — Workflows & Automations
**Date:** TBD (post Phase 1 closure)
**Previous work:** Phase 1 shipped. F-1 declared every Phase 2 `NodeKind` + `Trigger` variant in the enum from day 1 — the validator currently rejects them via `UnsupportedNodeKind { phase: 1 }`. This ticket unlocks Phase 2 variants for execution.

---

## What Is This Ticket?

Make the Phase 2 trigger + node kind variants **reachable** end-to-end without yet implementing their behaviour. Specifically:

1. Add the matching `NodeConfig::*` payload variants for `ToolCall`, `HttpRequest`, `ChannelMessage`, `Condition`, `Delay`. Phase 1 only had `NodeConfig::AgentPrompt`.
2. Bump `validator::allowed_node_kinds(2)` so the validator stops returning `UnsupportedNodeKind { phase: 1 }` for the Phase 2 kinds.
3. Add `executor::execute_node_dispatch(node)` that matches every `NodeConfig` variant and returns `Err(NotImplementedYet)` for the Phase 2 ones — the per-kind bodies land in F2-3..F2-7.
4. Update `agent::prompts::workflow_builder.md` to mention the new kinds + triggers in the schema-reminder block.
5. Add `Trigger::ChannelMessage` filter shape if F-1's stub left it incomplete.

### Why It Matters

- Every subsequent Phase 2 ticket depends on the scaffold + the validator unlock. Without this, F2-2..F2-11 can't even land tests.
- Locks the Phase 2 type vocabulary in one commit. F2-3..F2-7 fill bodies without re-defining types.

## Architectural Decisions

| Decision | Reference | Summary |
|---|---|---|
| Add variants — don't migrate | ADR-018 + F-1 | Serde tag-based enum is a schema-free extension |
| One scaffold ticket before per-kind impls | F-1 precedent | Locks the type vocabulary in one reviewable commit |
| `NotImplementedYet` errors instead of `unimplemented!()` | This ticket | Workflows referencing Phase 2 kinds save fine; runtime returns a structured error rather than panic |
