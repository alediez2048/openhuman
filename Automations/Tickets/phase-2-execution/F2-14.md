# F2-14 Primer: 30-Day Soft-Delete Retention Sweep

**For:** New coding-agent session
**Project:** OpenHuman — Workflows & Automations
**Previous work:** F-2 hard-deletes workflows via FK cascade today; F-15's WorkflowDeletePreview already surfaces `retention_days: 30`. The actual retention sweep is deferred from Phase 1.

---

## What Is This Ticket?

Convert the Phase 1 hard-delete into a soft-delete with a 30-day retention window. Background sweep purges aged rows. Per FR-1.3.4 + the deferred follow-up tracked in the Phase 1 README.

### Why It Matters

- The F-15 delete preview UI tells the user "run history is kept for 30 days after deletion" — but the actual cascade dropped it immediately. Users restoring a deleted workflow have no recourse today.

---

## What This Ticket Must Accomplish

### Goal

- Migration `005_workflow_soft_delete.sql`: add `workflows.deleted_at TIMESTAMP NULL`.
- `workflows_delete` (F-2): change from `DELETE` to `UPDATE workflows SET deleted_at = NOW()`. `workflow_runs` + `workflow_run_steps` stay around.
- `workflows_list` filter: exclude `deleted_at IS NOT NULL` by default. New optional `include_deleted: bool` field on `ListFilter`.
- New `workflows_restore(id)` RPC + agent tool: `workflow_propose_restore(id)` for the chat surface. UPDATE clears `deleted_at`.
- Background sweep `retention::run_purge_sweep(config)`:
    - Walks `workflows WHERE deleted_at < NOW() - 30 days`.
    - For each: hard-delete + cascade (F-2's original `DELETE FROM workflows`).
    - Publishes `WorkflowPurged { workflow_id, run_count }` per row.
- Schedule: tokio task fires the sweep once per hour. Documented + tested via a `now_provider` injection point so the test can fast-forward 31 days.
- F-9 orphan_recovery_sweep stays — but skips soft-deleted rows (they can't fire).

### Deliverables Checklist

#### A. Implementation

- [ ] Migration 005.
- [ ] `workflows::ops::delete` → soft-delete.
- [ ] `workflows::ops::restore` + new RPC + agent propose tool.
- [ ] `retention::run_purge_sweep` + tokio task in `jsonrpc.rs` boot.
- [ ] List filter `include_deleted: bool`.

#### B. Tests

- [ ] Soft-delete: row marked `deleted_at`; subsequent `workflows_list` excludes it.
- [ ] Restore returns it.
- [ ] Sweep with a fast-forwarded `now_provider` purges 31-day-old rows.
- [ ] FK cascade on hard-purge.

## Architectural Decisions

| Decision | Reference | Summary |
|---|---|---|
| Hourly sweep, not daily | This ticket | One purge/h is cheap; daily creates a thundering-herd window |
| Restore is its own propose tool | ADR-012 | Same single-mutation-boundary contract as delete |
| Run history retained until purge | FR-1.3.4 | Lets the user inspect why they deleted before the row's gone |
