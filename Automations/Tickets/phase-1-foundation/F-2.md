## What Is This Ticket?

Fill in the empty `ops.rs`, `rpc.rs`, and `schemas.rs` stubs created in F-1 with the full Phase 1 mutating + reading RPC surface for workflows. Implements: `workflows_list`, `workflows_get`, `workflows_create`, `workflows_update`, `workflows_delete`, `workflows_enable`, `workflows_disable`. Every mutating RPC publishes the corresponding `Workflow*` DomainEvent. The `WorkflowOrigin` discriminator is set on every `workflows_create` call based on which Phase 1 caller invoked it (UserChat / UserForm / Seed{template_id}).

**Not in this ticket:** `workflows_run_now`, `workflows_cancel_run`, `workflows_list_runs`, `workflows_get_run` ‚Äî those depend on the executor (F-7, F-8, F-9). `workflows_list_starter_templates` is F-5.

### Why It Matters

- This is the first ticket where the workflows domain is reachable from the UI / agent surface ‚Äî every later ticket (F-4 onwards) depends on `workflows_list`, `workflows_create`, etc. existing.
- Locks the `WorkflowOrigin` discriminator wiring so F-5 (catalog) and the chat-driven creation flow (F-12/F-14) can pass the correct provenance at create time.
- Establishes the `RpcOutcome<T>` return-type contract for the workflows domain ‚Äî matching `AGENTS.md`.

## Architectural Decisions

| Decision | Reference | Summary |
|---|---|---|
| Mutating RPCs publish events | NFR-2.4.1, ADR-012 | The mutation surface is the RPC layer; events are observability |
| `origin` is set by the caller, not inferred | ADR-018 | UI/agent/catalog each pass their own `WorkflowOrigin` |
| Initial `health` computed at create | ADR-017 | List view never sees an uncomputed row; subsequent changes recompute via F-3 subscriber |

## Suggested Implementation Pattern

**`src/openhuman/workflows/ops.rs` (extract):**

```rust
pub async fn create(
    ctx: &Ctx,
    payload: CreateWorkflowRequest,
) -> RpcOutcome<Workflow> {
    let now = chrono::Utc::now();
    let id  = uuid::Uuid::now_v7().to_string();
    let connections = ctx.connections_aggregator().list().await?;

    let mut workflow = Workflow {
        id: id.clone(),
        schema_version: 1,
        name: payload.name,
        description: payload.description,
        enabled: false,
        origin: payload.origin.clone(),
        health: WorkflowHealth::Ready,           // overwritten below
        trigger: payload.trigger,
        nodes: payload.nodes,
        edges: payload.edges,
        settings: payload.settings,
        created_at: now,
        updated_at: now,
        last_run_at: None,
    };

    // F-3 implements the real recompute; the stub returns Ready until F-3 lands.
    workflow.health = crate::openhuman::workflows::health::recompute(&workflow, &connections);

    ctx.workflows_store().insert_workflow(&workflow)?;
    crate::core::event_bus::publish_global(DomainEvent::WorkflowDefined {
        workflow_id: id.clone(),
        origin: payload.origin,
    });
    tracing::info!(target: "workflows", "[workflows-rpc] create id={id} origin={:?}", workflow.origin);

    RpcOutcome::ok(workflow)
}
```

**JSON-shape validation:** rely on Serde's `#[deny(unknown_fields)]` (add it to `CreateWorkflowRequest`) so a malformed payload fails at deserialization with a clean error, before reaching `ops::create`.

## Edge Cases to Handle

1. **`workflows_create` with an `id` already in the table.** UUIDv7 collisions are astronomically rare, but if `Uuid::now_v7()` does collide (clock skew across processes), return `RpcOutcome::Err { code: "conflict" }`.
2. **`workflows_update` on a deleted workflow.** Return `RpcOutcome::Err { code: "not_found" }`.
3. **`workflows_enable` on a `health != Ready` workflow.** The RPC succeeds (flips the bit) but the scheduler (F-7) will refuse to register the cron job. This matches FR-1.2.4 ‚Äî the UI is responsible for gating the toggle; the RPC stays permissive so chat-driven flows work.
4. **`workflows_disable` on an already-disabled workflow.** No-op; return `Ok(workflow)`. Don't publish a redundant `WorkflowDisabled` event (check the previous state first).
5. **Concurrent mutations on the same workflow.** SQLite serializes writes; the last writer wins. Document this ‚Äî Phase 1 doesn't add ETags.
6. **Cascade delete leaves orphaned event-bus subscribers.** Health-recomputation subscribers (F-3) use stored workflow ids; when the workflow is deleted they'll silently skip on the next event. Acceptable for Phase 1; F-15's capability-catalog DEVLOG documents this.
7. **`origin = Imported`** in a `CreateWorkflowRequest`. Phase 1 has no import path ‚Äî reject with `invalid_argument` so accidental clients can't forge the discriminator.
8. **Empty `nodes[]` array.** Reject with `invalid_argument` ‚Äî a workflow without nodes is meaningless. (The validator in F-11 enforces the same for proposals.)
9. **`description` is `Option<String>`.** Serde defaults to `None` if absent; that's fine.
