## What Is This Ticket?

Build the four chat-rich-message components that render the propose-tool payloads from F-12, and register them with `ChatRuntimeProvider`'s rich-message renderer.

**Design source of truth:** `Automations/Artifacts/designs/workflow-proposal-preview.md` (299 lines, locked). This ticket implements that design precisely. **Do not redesign in-flight.** If the design needs clarification, file a follow-up issue against the artifact rather than improvising.

The four components:
- `WorkflowProposalPreview.tsx` — the hero. Minimalist base + details disclosure + saved-state morph. Hosts Save / Save & Enable / Discard buttons that call `workflows_create` (+ `_enable`) directly.
- `WorkflowEditPreview.tsx` — diff-card pattern for `workflow_propose_update` payloads. Hosts Apply / Cancel buttons that call `workflows_update`.
- `WorkflowDeletePreview.tsx` — coral confirmation card. Hosts Delete / Cancel buttons that call `workflows_delete`.
- `WorkflowStatePreview.tsx` — tighter card with a single primary button for enable/disable/run-now confirmation.

Per ADR-020 (synthesis: minimalist + inspectable + conversational) and ADR-012 (clicks invoke RPCs directly; agent never mutates).

### Why It Matters

- F-14 closes the chat-driven creation loop. After F-14, the user can describe a workflow in chat, see the proposal preview, click Save, and find the workflow in `/workflows` — all without leaving the chat thread.
- Locks the rich-message renderer integration for workflows. Future preview components (Phase 2/3 schedule changes, etc.) reuse the same pattern.
- Implements the saved-state conversational morph (per ADR-020) — the design choice that makes the save feel like the agent kept talking.

| Decision | Reference | Summary |
|---|---|---|
| Synthesis design — minimalist base, inspectable disclosure, conversational morph | ADR-020 | Single visual language across create/edit/delete/state |
| Buttons invoke RPCs directly; agent never sees the click | ADR-010, ADR-012 | Structural security boundary |

## Suggested Implementation Pattern

**`app/src/components/workflows/preview/WorkflowProposalPreview.tsx` (extract):**

```tsx
type ViewState = 'pending' | 'saving' | 'saved' | 'error' | 'discarded';

export function WorkflowProposalPreview(props: WorkflowProposalPreviewProps) {
  const { proposal, threadId } = props;
  const t = useT();
  const actions = useWorkflowProposalActions(proposal, threadId);
  const [state, setState] = useState<ViewState>(props.state ?? 'pending');
  const [errorMessage, setErrorMessage] = useState<string | null>(null);
  const [expanded, setExpanded] = useState(false);
  const [savedMode, setSavedMode] = useState<'paused' | 'enabled' | null>(null);

  const handleSave = async (enable: boolean) => {
    setState('saving');
    try {
      await (enable ? actions.saveAndEnable() : actions.savePaused());
      setSavedMode(enable ? 'enabled' : 'paused');
      setState('saved');
    } catch (err: any) {
      setErrorMessage(err?.message ?? 'unknown');
      setState('error');
    }
  };

  if (state === 'saved') {
    return <SavedStub mode={savedMode!} workflowId={actions.lastWorkflowId} name={proposal.name} />;
  }
  if (state === 'discarded') {
    return <DiscardedStub onUndo={() => setState('pending')} />;
  }

  return (
    <div
      className="bg-white rounded-2xl shadow-soft border border-ocean-100 p-4 max-w-[560px]"
      aria-busy={state === 'saving'}
    >
      <ProposalHeader name={proposal.name} confidence={proposal.confidence} description={proposal.description} />
      <TriggerLine trigger={proposal.trigger} />
      <ConnectionChips required={proposal.required_connections} missing={proposal.missing_connections} />
      {proposal.setup_instructions && <SetupInstructionsCallout text={proposal.setup_instructions} />}
      {proposal.missing_connections.length > 0 && <MissingConnectionsBanner missing={proposal.missing_connections} />}
      <DetailsDisclosure expanded={expanded} onToggle={() => setExpanded(v => !v)} />
      {expanded && <DetailsPanel proposal={proposal} />}
      <ActionRow
        state={state}
        errorMessage={errorMessage}
        canSaveEnable={proposal.missing_connections.length === 0}
        onDiscard={() => setState('discarded')}
        onSavePaused={() => handleSave(false)}
        onSaveAndEnable={() => handleSave(true)}
      />
    </div>
  );
}
```

**`useWorkflowProposalActions` hook:**

```ts
export function useWorkflowProposalActions(proposal: WorkflowProposal, threadId: string) {
  const lastWorkflowIdRef = useRef<string | null>(null);

  const savePaused = async () => {
    const created = await workflowsApi.create({ ...proposalToCreateRequest(proposal), origin: { type: 'user_chat' } });
    lastWorkflowIdRef.current = created.id;
    await coreRpcClient.call('chat.append_user_message', {
      thread_id: threadId,
      content: `Saved as '${proposal.name}'.`,
    });
  };

  const saveAndEnable = async () => {
    await savePaused();
    if (lastWorkflowIdRef.current) {
      await workflowsApi.enable(lastWorkflowIdRef.current);
    }
  };

  return { savePaused, saveAndEnable, get lastWorkflowId() { return lastWorkflowIdRef.current; } };
}
```

## Edge Cases to Handle

1. **Concurrent click on Save (paused) and Save & Enable.** Use a single `loading` state; second button is disabled while the first is in flight.
2. **`workflows_create` succeeds but `workflows_enable` fails.** State transitions to `error` with the message "Saved, but couldn't enable: <reason>. Toggle it manually from /workflows." Don't roll back the create.
3. **Discard then Undo within 15s.** `discarded` state's Undo button transitions back to `pending`. Save state is fresh — the original proposal still in props.
4. **Chat thread loses context** (page refresh between proposal and save). The chat history is immutable — re-rendering shows the proposal in its terminal state (which is pending until clicked). Document.
5. **`coreRpcClient.call('chat.append_user_message', ...)` fails.** The save succeeded; the synthetic message just doesn't appear. Log to console; don't error the save.
6. **Long workflow name.** Truncate in the header with `title` for full text. The saved stub shows the full name (no truncation there).
7. **Many connections** (>10 chips). Wrap; constraint to `max-h-24 overflow-hidden` and add a "+N more" pill.
8. **`prefers-reduced-motion`.** Skip the saved-state morph animation; jump directly to the stub.
9. **`useCronHumanizer` doesn't recognize a Phase 2 trigger.** Fallback "Custom trigger" string.
10. **A11y: keyboard-only navigation through the details panel.** All sections reachable via Tab; collapse keys work.
11. **Edit-preview diff with no bullets** (current === proposed). Don't render the diff section; show a message "No changes proposed." Apply button disabled.
12. **Delete-preview with `run_count = 0`.** Pluralization: "0 past runs" → "no past runs."
