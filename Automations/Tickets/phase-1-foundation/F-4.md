## What Is This Ticket?

First frontend ticket of Phase 1. Adds the dedicated 8th bottom-tab between **Connections** and **Intelligence** (locked OQ-1 = A), the `/workflows` route, and the list-view page with `WorkflowsList`, `WorkflowCard`, `WorkflowHealthBadge`, and the empty state. The card's primary action is the **enable/disable toggle** wired to `workflows_enable` / `workflows_disable`. Empty state CTAs: "Ask OpenHuman to build a workflow" (top) and a placeholder "Starter workflows" section (F-5 populates it).

**Not in this ticket:** the rich preview components for chat-driven creation (F-14), the starter-catalog section's actual content (F-5), the run-history table (deferred to a follow-up since `workflows_list_runs` lands in F-8/F-9), the form-fallback create modal (low-emphasis per FR-1.2.6 — can ship in F-15 or a follow-up).

### Why It Matters

- This is the first user-visible surface of Phase 1. Without it, none of the backend work from F-1..F-3 is reachable.
- Locks the nav placement decision (OQ-1 = A) into the bottom-tab order: Home · Human · Chat · Connections · **Workflows** · Intelligence · Alerts · Rewards · Settings. The order is informationally meaningful — connections (inputs) sit just left of workflows (compositions).
- Establishes the `workflowsSlice` and `services/api/workflows.ts` client every subsequent frontend ticket reuses.

## Suggested Implementation Pattern

**`app/src/components/workflows/WorkflowCard.tsx` (extract):**

```tsx
export function WorkflowCard({ workflow }: { workflow: Workflow }) {
  const t = useT();
  const dispatch = useAppDispatch();
  const healthy = workflow.health.type === 'ready';
  const toggleLabel = workflow.enabled ? t('workflows.disable') : t('workflows.enable');

  return (
    <div className="bg-white rounded-2xl shadow-soft border border-ocean-100 p-4 flex items-center gap-4">
      <div className="flex-1 min-w-0">
        <div className="flex items-center gap-2">
          <h3 className="font-semibold text-stone-900 truncate">{workflow.name}</h3>
          <WorkflowHealthBadge health={workflow.health} />
        </div>
        <div className="text-sm text-stone-500 mt-1">
          {summarizeTrigger(workflow.trigger, t)} · {summarizeNodes(workflow.nodes, t)}
        </div>
      </div>
      <WorkflowEnableToggle
        workflowId={workflow.id}
        enabled={workflow.enabled}
        disabledReason={!healthy ? t('workflows.toggle_disabled_health', { reason: missingLabel(workflow.health) }) : undefined}
        onToggle={(next) => dispatch(next ? enableWorkflow(workflow.id) : disableWorkflow(workflow.id))}
      />
      <OverflowMenu items={[t('workflows.edit'), t('workflows.run_now'), t('workflows.delete')]} />
    </div>
  );
}
```

## Edge Cases to Handle

1. **`workflows_list` returns an error.** Show a `t('workflows.list_error')` banner above the list and a Retry button. Don't crash the page.
2. **`workflows_enable` rejects (e.g., race with delete).** Slice action records the error; UI shows a transient toast; the toggle visually reverts. Don't swallow the error.
3. **Workflow with `last_run_at = null`.** Trigger summary shows "Never run" / "Next run: <ts>" depending on whether the trigger is cron-with-tz or manual.
4. **Workflow with an unknown `trigger.type`** (forward-compat with Phase 2 triggers shipped early). Render a fallback "Custom trigger" label; don't crash.
5. **Toggle race conditions.** Disable the toggle while the thunk is `pending`. Re-enable once the fulfilled/rejected dispatches.
6. **Long workflow names.** Truncate with `truncate` Tailwind class + show full name in `title` attribute for accessibility.
7. **Bottom-tab visual ordering on different viewports.** The new tab might push the layout if it overflows. Verify on Linux + macOS at 1280px and 1920px widths.
8. **i18n missing key fallback.** If a locale chunk is missing `nav.workflows` (forgot one), the fallback in `I18nContext.tsx` shows the key string — visible bug, but non-crashing. The test catches this by asserting the resolved string per locale.
9. **`hideStarterSection` redux-persist.** Mark only this field as persisted (not `workflows` itself, which must always re-fetch on visit).
