## What Is This Ticket?

Register the four read-only workflow agent tools that F-8's `build_node_agent_definition` references by name, and add the exhaustive unit test that satisfies Phase 1 DoD: the returned allowlist contains exactly baseline + allowed_connections + 4 read-only workflow tools — and zero `workflow_propose_*` or mutating tools.

Tools to register: `workflow_list`, `workflow_get`, `workflows_list_runs`, `workflows_get_run`. All READ-ONLY — no mutating tools land on the agent surface this ticket. (F-12 lands the propose-only tools.)

### Why It Matters

- Without F-10, the agent runtime in `agent_prompt` nodes (F-8) references tool names that don't actually exist — calls would fail at runtime.
- The allowlist enforcement test is a Phase 1 DoD requirement (NFR-2.6.5, Phase 1 acceptance §5 line "agent_prompt sub-agent tool allowlist matches NFR-2.3.7"). Without this test, the security boundary from ADR-016 is unverified.
- Locks the read-only-only convention for the workflows agent surface. Every Phase 2 expansion adds tools through the same pattern.

## Suggested Implementation Pattern

**`src/openhuman/workflows/agent_tools.rs` (extract):**

```rust
const TOOL_WORKFLOW_LIST:       &str = "workflow_list";
const TOOL_WORKFLOW_GET:        &str = "workflow_get";
const TOOL_WORKFLOWS_LIST_RUNS: &str = "workflows_list_runs";
const TOOL_WORKFLOWS_GET_RUN:   &str = "workflows_get_run";

pub fn read_only_tool_names() -> &'static [&'static str] {
    &[
        TOOL_WORKFLOW_LIST,
        TOOL_WORKFLOW_GET,
        TOOL_WORKFLOWS_LIST_RUNS,
        TOOL_WORKFLOWS_GET_RUN,
    ]
}

pub async fn handle_workflow_list(ctx: &Ctx, args: WorkflowListArgs) -> serde_json::Value {
    tracing::info!(target: "workflows-agent", "tool=workflow_list filter={:?}", args.filter);
    let workflows = ops::list(ctx, args.filter.unwrap_or_default()).await.into_json();
    workflows
}

pub fn register_all(registry: &mut Registry) {
    registry.register(Tool {
        name: TOOL_WORKFLOW_LIST.into(),
        description: "List the user's workflows. Read-only.".into(),
        schema: workflow_list_schema(),
        handler: Arc::new(|ctx, args| Box::pin(handle_workflow_list(ctx, args))),
    });
    // ... three more ...
}
```

**Allowlist negative assertion (`agent_tools_tests.rs`):**

```rust
#[tokio::test]
async fn no_mutating_workflow_tools_registered() {
    let mut registry = Registry::new();
    workflows::agent_tools::register_all(&mut registry);
    let names: Vec<String> = registry.list_tools();

    // Positive: the four read tools are present.
    for expected in workflows::agent_tools::read_only_tool_names() {
        assert!(names.contains(&expected.to_string()), "missing tool: {expected}");
    }

    // Negative: zero propose tools.
    let propose_count = names.iter().filter(|n| n.starts_with("workflow_propose_")).count();
    assert_eq!(propose_count, 0, "propose tools must not be registered in F-10: {names:?}");

    // Negative: zero mutation RPC names.
    let forbidden = [
        "workflows_create", "workflows_update", "workflows_delete",
        "workflows_enable", "workflows_disable", "workflows_run_now",
        "workflows_cancel_run", "workflow_create_from_proposal",
    ];
    for f in forbidden {
        assert!(!names.iter().any(|n| n == f), "mutating name leaked: {f}");
    }
}

#[tokio::test]
async fn build_node_agent_definition_matches_nfr_2_3_7() {
    let allowed = vec![ConnectionRef::Composio { toolkit_id: "gmail".into(), account_id: None }];
    let definition = executor::build_node_agent_definition(&allowed);

    let baseline = agent::baseline_tools();
    for b in &baseline {
        assert!(definition.allowed_tools.contains(b), "missing baseline tool: {b}");
    }
    for t in workflows::agent_tools::read_only_tool_names() {
        assert!(definition.allowed_tools.contains(&t.to_string()),
            "missing read-only workflow tool: {t}");
    }
    let propose_count = definition.allowed_tools.iter()
        .filter(|n| n.starts_with("workflow_propose_")).count();
    assert_eq!(propose_count, 0, "propose tool leaked into agent_prompt allowlist");
}
```

## Edge Cases to Handle

1. **Tool name typos vs F-8's references.** F-10's `read_only_tool_names()` constant is the canonical source; F-8 should use these constants (refactor F-8 if it hard-coded strings).
2. **`workflows_list_runs` argument validation.** Reject pagination with `limit > 100`; agent tools are easier to misuse than RPCs. Return a structured JSON error rather than crashing.
3. **Empty workflow list.** Return `[]`, not null.
4. **Run not found in `workflows_get_run`.** Return `null` (not an error) so the agent can reason "no run with that id."
5. **Tool registration order.** Must run after `WorkflowsStore::open` (the handlers consume `ctx.workflows_store()`). Verify the boot order is correct.
6. **Sensitive fields in `workflow_get` output.** No secrets live in workflow JSON (per NFR-2.3.2), but agent prompt text might be long — keep the response shape lean. Document.
7. **`tools::registry::list_tools()` API.** If the existing API doesn't expose tool names (e.g., it returns full `Tool` structs), use `.iter().map(|t| t.name.clone()).collect::<Vec<_>>()`. Don't add a new API surface for the test — wrap inline.
8. **Test isolation.** The allowlist test mustn't depend on global state; instantiate a fresh `Registry::new()` in each test, register only the workflows tools, and assert against the local list. (If the codebase uses a singleton registry, document the workaround.)
