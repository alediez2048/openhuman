## Suggested Implementation Pattern

**`src/openhuman/workflows/proposer.rs` (relevant section):**

```rust
const WORKFLOW_BUILDER_PROMPT: &str = include_str!("../agent/prompts/workflow_builder.md");

fn build_system_prompt(
    connections: &ConnectionsSnapshot,
    phase: u32,
    last_error: Option<&ProposalValidationError>,
) -> String {
    let mut out = String::with_capacity(WORKFLOW_BUILDER_PROMPT.len() + 1024);
    out.push_str(WORKFLOW_BUILDER_PROMPT);
    out.push_str("\n\n## Your connections\n\n");
    out.push_str(&summarize_connections(connections));
    out.push_str("\n\n## Phase ");
    out.push_str(&phase.to_string());
    out.push_str(" constraints\n\n");
    out.push_str(&phase_constraints_block(phase));
    if let Some(e) = last_error {
        out.push_str("\n\n## PREVIOUS ATTEMPT FAILED\n\n");
        out.push_str(&format_validation_error(e));
    }
    out
}

fn summarize_connections(snapshot: &ConnectionsSnapshot) -> String {
    if snapshot.is_empty() {
        return "(no connections yet)".to_string();
    }
    let mut lines = Vec::new();
    let composio = snapshot.composio_toolkits();
    if !composio.is_empty() {
        lines.push(format!("- Composio: {}", composio.join(", ")));
    }
    let channels = snapshot.channel_providers();
    if !channels.is_empty() {
        lines.push(format!("- Channel: {}", channels.join(", ")));
    }
    let webviews = snapshot.webview_providers();
    if !webviews.is_empty() {
        lines.push(format!("- Webview: {}", webviews.join(", ")));
    }
    // ... built-in, MCP, generic HTTP ...
    lines.join("\n")
}

fn phase_constraints_block(phase: u32) -> String {
    match phase {
        1 => "Triggers: `cron`, `manual`.\nNode kinds: `agent_prompt` (exactly one node per workflow).".into(),
        2 => "Triggers: `cron`, `manual`, `webhook`, `composio_event`, `channel_message`.\nNode kinds: `agent_prompt`, `tool_call`, `http_request`, `channel_message`, `condition`, `delay`.".into(),
        _ => "All Phase 1+2 triggers + node kinds. Phase 3 visual canvas adds reserved kinds.".into(),
    }
}
```

## Edge Cases to Handle

1. **`workflow_builder.md` not found at compile time.** `include_str!` fails the build — that's the point. Verify the path before committing.
2. **Tauri resource path syntax.** The `resources` array uses paths relative to `app/src-tauri/`. Match the existing prompt entries exactly; don't guess the resolution.
3. **Prompt content changes after F-13.** Future updates edit `Automations/Artifacts/prompts/workflow_builder.md` (design-time SoT) AND `src/openhuman/agent/prompts/workflow_builder.md` (production). The DEVLOG entry should note this dual-write expectation. Long term, a follow-up could symlink; for now, dual-maintain.
4. **`summarize_connections` with > 20 entries per mechanism.** Cap each line at 10 names + "(and N more)". Keeps the prompt under the model's context budget.
5. **Phase-3+ phase block.** Default to "all known triggers and node kinds" so the function never panics on an unknown phase.
6. **`format_validation_error` format.** Use the structured `kind_label` + a short message; mirror `systemsdesign.md §4.4`'s "PREVIOUS ATTEMPT FAILED" example.
7. **Bundle path divergence between Linux / macOS / Windows.** Tauri normalizes; verify on the OS you're developing on. CI runs Linux only — make sure CI doesn't regress.
8. **Hot-reload during dev.** `include_str!` is compile-time only — editing the prompt requires `cargo build` again. Document.
