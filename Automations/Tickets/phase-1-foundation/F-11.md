## What Is This Ticket?

Implement `proposer::draft_with_retries(description, connections_snapshot, phase, max_attempts=3)` — the bounded-auto-retry workflow-proposal-drafting sub-agent — and the `validator::validate(proposal, connections, phase)` function it consults.

Each retry attempt runs a drafting sub-agent via `agent::run_subagent` with the `workflow_builder.md` system prompt + the connections snapshot inlined + a tight tool allowlist (`connections_list`, `workflow_list`, synthetic `emit_proposal`). The drafting sub-agent's output is validated; on validation failure, the structured `ProposalValidationError` is appended to the system prompt for the next attempt. After 3 failures, return `DraftFailure::ValidationFailedAfterRetries`.

Per ADR-015 (bounded retry budget = 3) and ADR-019 (structured error enum).

### Why It Matters

- This is the engine of the chat-driven hero flow. Every `workflow_propose_create` call from F-12 ultimately funnels through `draft_with_retries`.
- The validator IS the safety boundary that converts "the LLM wrote some JSON" into "a workflow that can be safely persisted." Every Phase 1 unit test under NFR-2.6.5 hits this code.
- The drafting sub-agent's tool allowlist is DIFFERENT from `agent_prompt`'s allowlist (per ADR-016) — it has `connections_list` + `workflow_list` + the synthetic `emit_proposal`, and nothing else. F-11 wires this allowlist; F-10 wired the `agent_prompt` one.

## What This Ticket Must Accomplish

### Goal

- `validator::validate(proposal: &WorkflowProposal, connections: &ConnectionsSnapshot, phase: u32) -> Result<(), ProposalValidationError>` implementing every check from FR-1.13.9:
    - JSON deserializes into `WorkflowProposal`. (Caller deserializes; validator validates the deserialized shape — split is `JsonParse` happens at the deserialization step.)
    - `required_connections` ⊆ `connections_list` output.
    - `nodes[].kind` ∈ phase-allowed set (Phase 1: only `AgentPrompt`).
    - `cron` expressions parse via the `cron` crate.
    - Edge integrity: every edge's `from` / `to` references a node id that exists in `nodes`.
    - `agent_prompt.allowed_connections` are all known to `connections`.
    - `MissingRequiredField` checks: `name` non-empty, `description` non-empty, `nodes` non-empty.
- For each validation failure, return the corresponding `ProposalValidationError` variant. Validation must run in < 50ms (NFR-2.1.5) — fast deterministic checks, no LLM calls.
- For `UnknownConnection` errors, compute a `candidates: Vec<ConnectionRef>` list using fuzzy matching (Levenshtein distance ≤ 3, or a simpler prefix-overlap heuristic) against the user's actual connection names. Limit to 3 candidates.
- `proposer::draft_with_retries(description, connections, phase, max_attempts)`:
    1. For attempt 0..max_attempts:
        - Build the system prompt via `build_system_prompt(connections, phase, last_error)`.
        - Run the drafting sub-agent: `agent::run_subagent(definition, &description, parent_context).await`.
        - Extract the JSON payload from the sub-agent's `emit_proposal` tool call.
        - Validate; if Ok, return the proposal.
        - On error: increment `metrics::counter!("workflow_proposal_validation_error", "kind" => e.kind_label())`; store `last_error = Some(e)`; continue.
    2. After max_attempts failures, return `Err(DraftFailure::ValidationFailedAfterRetries { attempts, last_error })`.
- `proposer::build_system_prompt(connections, phase, last_error) -> String`:
    - Loads `src/openhuman/agent/prompts/workflow_builder.md` via `include_str!` (the file lives in this location after F-13 lands; if F-13 hasn't merged, place a placeholder file with the content from `Automations/Artifacts/prompts/workflow_builder.md` so F-11 can compile and test).
    - Templates in the connections summary (e.g., "You have these connections: Composio: gmail, slack, linear. Webview: linkedin, twitter.").
    - Templates in phase-specific schema reminders (Phase 1: `Trigger::Cron`, `Trigger::Manual`; `NodeKind::AgentPrompt` only).
    - Appends `last_error` as a "PREVIOUS ATTEMPT FAILED:" block when present (per `systemsdesign.md §4.4`).
- The drafting sub-agent's tool allowlist (different from agent_prompt's): `connections_list`, `workflow_list`, `emit_proposal`. **Iteration cap default 6** per FR-1.13.2.
- `emit_proposal(payload)` is a synthetic tool: the wrapper around `run_subagent` extracts the payload from the tool call and short-circuits — it does NOT execute as a "real" tool. Implement via a `Sentinel` enum the wrapper inspects.

## Architectural Decisions

| Decision | Reference | Summary |
|---|---|---|
| Bounded retry budget = 3 | ADR-015 | Predictable 90s worst case (NFR-2.1.4) |
| Validator is deterministic; never LLM-driven | ADR-019 | < 50ms guarantee (NFR-2.1.5) |
| Drafting sub-agent has its own (tighter) allowlist | ADR-016 / ADR-009 | `connections_list` + `workflow_list` + `emit_proposal` only |
| `emit_proposal` is a synthetic tool (not in tools::registry) | systemsdesign §4.2 | The wrapper intercepts the tool call; the payload doesn't round-trip through registry |
| Fuzzy candidates for `UnknownConnection` | ADR-019 | Surgical retry prompts; helps the agent correct typos |
| System prompt loaded via `include_str!` from `agent/prompts/workflow_builder.md` | NFR-2.5.6 | Same convention as other agent prompts |

## Suggested Implementation Pattern

**`src/openhuman/workflows/proposer.rs` (extract):**

```rust
pub async fn draft_with_retries(
    ctx: &Ctx,
    description: &str,
    connections: &ConnectionsSnapshot,
    phase: u32,
    max_attempts: u32,
) -> Result<WorkflowProposal, DraftFailure> {
    let mut last_error: Option<ProposalValidationError> = None;
    for attempt in 0..max_attempts {
        tracing::info!(target: "workflows-proposer",
            "attempt {}/{} for description={}", attempt + 1, max_attempts,
            &description.chars().take(80).collect::<String>());
        let system_prompt = build_system_prompt(connections, phase, last_error.as_ref());
        let proposal = match run_drafting_subagent(ctx, description, system_prompt).await {
            Ok(p) => p,
            Err(e) => return Err(DraftFailure::RunFailure { reason: e.to_string() }),
        };
        match validator::validate(&proposal, connections, phase) {
            Ok(()) => return Ok(proposal),
            Err(e) => {
                metrics::counter!(
                    "workflow_proposal_validation_error",
                    "kind" => e.kind_label()
                ).increment(1);
                tracing::warn!(target: "workflows-validator",
                    "attempt {} failed: {:?}", attempt + 1, e.kind_label());
                last_error = Some(e);
            }
        }
    }
    Err(DraftFailure::ValidationFailedAfterRetries {
        attempts: max_attempts,
        last_error: last_error.expect("loop ran ≥ once"),
    })
}

fn build_system_prompt(
    connections: &ConnectionsSnapshot,
    phase: u32,
    last_error: Option<&ProposalValidationError>,
) -> String {
    const BASE: &str = include_str!("../agent/prompts/workflow_builder.md");
    let summary = summarize_connections(connections);
    let phase_block = phase_constraints_block(phase);
    let mut prompt = BASE.to_string();
    prompt.push_str(&format!("\n\n## Your connections\n\n{summary}\n"));
    prompt.push_str(&format!("\n## Phase {phase} constraints\n\n{phase_block}\n"));
    if let Some(e) = last_error {
        prompt.push_str(&format!("\n## PREVIOUS ATTEMPT FAILED\n\n{}\n",
            format_validation_error(e)));
    }
    prompt
}
```

**Validator entry (`validator.rs`):**

```rust
pub fn validate(
    proposal: &WorkflowProposal,
    connections: &ConnectionsSnapshot,
    phase: u32,
) -> Result<(), ProposalValidationError> {
    if proposal.name.trim().is_empty() {
        return Err(ProposalValidationError::MissingRequiredField { field: "name".into() });
    }
    if proposal.nodes.is_empty() {
        return Err(ProposalValidationError::MissingRequiredField { field: "nodes".into() });
    }

    if let Trigger::Cron { expr, .. } = &proposal.trigger {
        if let Err(e) = cron::Schedule::from_str(expr) {
            return Err(ProposalValidationError::InvalidCron {
                expr: expr.clone(),
                parse_error: e.to_string(),
            });
        }
    }

    let allowed = allowed_node_kinds(phase);
    for node in &proposal.nodes {
        if !allowed.contains(&node.kind) {
            return Err(ProposalValidationError::UnsupportedNodeKind {
                kind: node.kind,
                phase,
            });
        }
    }

    for r in &proposal.required_connections {
        if !connections.contains(r) {
            return Err(ProposalValidationError::UnknownConnection {
                r#ref: r.clone(),
                candidates: fuzzy_candidates(r, connections),
            });
        }
    }

    // ... edge integrity, allowed_connections walk ...
    Ok(())
}
```

## Edge Cases to Handle

1. **Drafting sub-agent never calls `emit_proposal`** (hits iteration cap without emitting). Treat as `RunFailure { reason: "no proposal emitted" }`; retry counts.
2. **Drafting sub-agent emits invalid JSON to `emit_proposal`.** Catch at the wrapper layer with `serde_json::from_value`; return `ProposalValidationError::JsonParse { reason }`.
3. **`UnknownConnection` candidates are empty.** Return empty `candidates`; the agent might just need to drop the connection from required.
4. **Cron with timezone field that the cron crate doesn't validate.** The `cron` crate only validates the expression; document that `tz` is passed through and rendered in the trigger summary but not validated server-side.
5. **`EdgeIntegrity`** with Phase 1 zero edges. If `edges.is_empty()`, integrity check is vacuously true. Test this.
6. **`allowed_connections` references a `ConnectionRef` variant the validator hasn't seen** (e.g., a future Mcp variant). Use Serde's exhaustive matching; fail fast with `UnknownConnection`.
7. **Sub-agent times out** (mock LLM hangs). `agent::run_subagent` has its own timeout; respect that. The 90s worst-case from NFR-2.1.4 includes 3 attempts; cap each attempt to 30s.
8. **Validator throws on weirdly-formed `WorkflowProposal`.** Use `serde_json::from_value` in the proposer's tool-output parsing to fail early with `JsonParse`.
9. **System-prompt size budget.** The connections summary + phase block + base prompt + last-error block all fit in the model's context. Phase 1 base prompt is ~4k tokens; connections summary is bounded by user's connection count (~50 lines max); total stays comfortably under 8k.
10. **The `workflow_builder.md` file not yet on disk** (F-13 hasn't landed). Land a placeholder file at `src/openhuman/agent/prompts/workflow_builder.md` containing the content from `Automations/Artifacts/prompts/workflow_builder.md`. F-13 then verifies the tauri bundling and the file path.
