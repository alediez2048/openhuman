## Suggested Implementation Pattern

**`app/test/e2e/specs/workflows-agent-creation.spec.ts` (extract):**

```ts
import { describe, it, expect } from 'vitest'; // or wdio's native runner
import {
  clickNativeButton, waitForWebView, typeIntoElement,
  resetMockBackend, seedConnection, callCoreRpc,
} from '../helpers/element-helpers';

describe('Workflows — agent-driven creation (hero flow, NFR-2.6.3)', () => {
  it('chat → propose → save & enable → run → completes', async () => {
    await resetMockBackend();
    await seedConnection({ type: 'channel', provider: 'telegram', channel_id: 'fake-tg-123' });

    await waitForWebView();
    await clickNativeButton('nav-workflows');
    await expect($('=Ask OpenHuman to build a workflow')).toBeDisplayed();
    await clickNativeButton('=Ask OpenHuman to build a workflow');

    await typeIntoElement('chat-input',
      'every weekday at 8am, send me a Telegram message saying good morning.');
    await clickNativeButton('chat-send');

    const preview = await $('[data-testid="workflow-proposal-preview"]');
    await preview.waitForDisplayed({ timeout: 90_000 }); // NFR-2.1.4 worst case
    await clickNativeButton('=Save & Enable');

    await $('[data-testid="workflow-saved-stub"]').waitForDisplayed();
    await clickNativeButton('nav-workflows');
    const rows = await $$('[data-testid="workflow-card"]');
    expect(rows.length).toBe(1);

    const workflows = await callCoreRpc('workflows_list', {});
    const wfId = workflows[0].id;
    const { run_id } = await callCoreRpc('workflows_run_now', { workflow_id: wfId });

    let run;
    for (let i = 0; i < 60; i++) {
      run = await callCoreRpc('workflows_get_run', { run_id });
      if (run.status !== 'running') break;
      await browser.pause(1000);
    }
    expect(['succeeded', 'failed']).toContain(run.status);
    expect(run.steps.length).toBeGreaterThan(0);
  });
});
```

**`app/test/e2e/specs/workflows-seeded.spec.ts` (extract):**

```ts
describe('Workflows — starter catalog flow (NFR-2.6.4)', () => {
  it('catalog dedup add+delete cycle', async () => {
    await resetMockBackend();
    await waitForWebView();
    await clickNativeButton('nav-workflows');

    const initialCards = await $$('[data-testid^="starter-template-card-"]');
    expect(initialCards.length).toBe(4);

    await clickNativeButton('starter-template-card-ru-1-founder-morning-digest-add');
    await browser.waitUntil(async () =>
      (await $$('[data-testid^="starter-template-card-"]')).length === 3
    );
    const yourRows = await $$('[data-testid="workflow-card"]');
    expect(yourRows.length).toBe(1);

    const workflows = await callCoreRpc('workflows_list', {});
    await callCoreRpc('workflows_delete', { workflow_id: workflows[0].id });
    await browser.waitUntil(async () =>
      (await $$('[data-testid^="starter-template-card-"]')).length === 4
    );
  });
});
```

## Edge Cases to Handle

1. **Drafting sub-agent takes the full 90s.** Hero spec timeout is generous (90s for preview render); document NFR-2.1.4.
2. **Run never reaches terminal status** (e.g., mock LLM hangs). Poll budget is 60s after `workflows_run_now`. Spec fails with a clear "run did not complete in 60s" message.
3. **Telegram connection seed fails.** Spec fails early with a clear error — don't proceed to chat.
4. **Catalog spec races** (UI re-render lag between click and assertion). Use `waitUntil` rather than fixed `pause`.
5. **`workflows_delete` from the overflow menu is broken** (regression). Fall back to direct-RPC delete. Document the regression in the F-15 PR description and file a follow-up.
6. **`data-testid` attributes don't exist on every component.** Add them as part of F-15 — small, scoped edits to F-4 / F-6 / F-14 components. Note in DEVLOG.
7. **About_app capability shape differs from the sample above.** Read the actual file first; match its shape exactly.
8. **An ADR's decision was overridden during implementation.** Document explicitly in DEVLOG: "ADR-XX: spec says X, code implements Y because Z. Follow-up issue: #N."
9. **CI Linux vs macOS divergence.** The hero spec's chat→propose path involves the LLM — ensure the mock backend's LLM responses are deterministic on both platforms.
10. **Mock-server `__admin/reset` between specs.** Each spec's `resetMockBackend` call clears workflows.db + connections.db state. Verify the reset path covers the workflows tables.
